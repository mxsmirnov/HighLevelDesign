# Логическое представление (Logical View)

Данный раздел описывает логическую архитектуру системы. Элементы сгруппированы по типам: действующие лица, границы, сущности и элементы управления. Это позволяет понять, как функциональные требования (варианты использования) преобразуются в ключевые объекты системы.

## 1. Действующие лица (Actors)
Роли пользователей или внешних систем, взаимодействующих с системой.

| Actor | Описание | Примеры |
| :--- | :--- | :--- |
| **Пользователь** | Любое лицо, использующее систему. | Любой посетитель сайта. |
| **Зарегистрированный пользователь** | Пользователь, прошедший процедуру аутентификации. | Иван Петров (клиент интернет-магазина). |
| **Администратор** | Пользователь с привилегиями на управление контентом и пользователями. | Сотрудник, управляющий каталогом товаров. |
| **Внешняя платежная система** | Внешний сервис, обрабатывающий платежи. | API платежного шлюза (например, Stripe, ЮKassa). |
| **Библиотекарь** | Сотрудник библиотеки, управляющий выдачей книг (для системы библиотеки). | Сотрудник абонемента. |

## 2. Варианты использования (Use Case)
Функциональные требования, описывающие, как акторы взаимодействуют с системой для достижения конкретной цели. Каждый вариант использования может быть детализирован через элементы ниже.

| ID | Название | Основной актор | Краткое описание |
| :--- | :--- | :--- | :--- |
| **UC-01** | Оформить заказ | Зарегистрированный пользователь | Пользователь выбирает товары из корзины, указывает адрес доставки и оплачивает заказ. |
| **UC-02** | Найти книгу | Пользователь, Библиотекарь | Поиск книги в каталоге по названию, автору или ключевым словам. |
| **UC-03** | Зарегистрироваться | Пользователь | Создание новой учетной записи в системе. |
| **UC-04** | Добавить товар | Администратор | Добавление новой позиции в каталог интернет-магазина. |

## 3. Границы (Boundaries)
Элементы интерфейса, через которые акторы взаимодействуют с системой (формы, экраны, точки доступа API).

| Boundary (Интерфейс) | Связанный Use Case | Описание | Элементы интерфейса (примеры) |
| :--- | :--- | :--- | :--- |
| **Форма входа** | UC-03 | Интерфейс для аутентификации пользователя. | Поля "Логин", "Пароль", кнопка "Войти". |
| **Страница каталога** | UC-02 | Экран отображения списка книг/товаров с фильтрами. | Строка поиска, фильтры по жанру/цене, карточки товаров. |
| **Форма оформления заказа** | UC-01 | Пошаговый интерфейс для ввода данных доставки и оплаты. | Поля "Адрес", "Способ доставки", кнопка "Оплатить". |
| **API Эндпоинт /api/orders** | UC-01 | Точка входа для передачи данных о заказе из мобильного приложения или фронтенда. | JSON-структура с товарами и данными покупателя. |
| **Панель администратора (форма товара)** | UC-04 | Форма для ввода характеристик нового товара. | Поля "Название", "Цена", "Описание", кнопка "Сохранить". |

### 4. Сущности (Entities)
Объекты предметной области, которые хранят данные и имеют состояние. Обычно соответствуют таблицам в базе данных.

| Entity | Атрибуты (примеры) | Поведение / Методы | Отношения |
| :--- | :--- | :--- | :--- |
| **Пользователь (User)** | ID, Имя, Email, Хеш пароля, Роль | register(), login(), updateProfile() | Имеет много **Заказов**. |
| **Товар (Product)** | ID, Артикул, Название, Цена, Остаток | updateStock(), getDetails() | Принадлежит **Категории**. |
| **Заказ (Order)** | ID, Дата, Статус, Сумма, ID пользователя | calculateTotal(), changeStatus() | Принадлежит **Пользователю**, содержит много **Товаров** (через OrderItem). |
| **Книга (Book)** | ISBN, Название, Автор, Год издания, Статус (в библиотеке/выдана) | markAsBorrowed(), markAsReturned() | Может быть связана с **Читательским билетом**. |

### 5. Элементы решения (Controls)
Логика, управляющая сценариями, координирующая работу границ и сущностей. Содержат бизнес-правила конкретного варианта использования.

| Control (Контроллер/Менеджер) | Реализуемый Use Case | Координирует | Описание логики |
| :--- | :--- | :--- | :--- |
| **Контроллер аутентификации** | UC-03 | Форма входа, Сущность **Пользователь** | Получает данные из формы, ищет пользователя в БД, проверяет пароль, создает сессию. |
| **Менеджер поиска** | UC-02 | Поля ввода, Сущность **Книга/Товар** | Принимает поисковый запрос, обращается к поисковому индексу или БД, возвращает отфильтрованный список сущностей для отображения на границе. |
| **Процессор заказов** | UC-01 | Форма заказа, Сущности **Заказ**, **Товар**, **Пользователь**, Внешняя платежная система | Проверяет остатки товаров, создает сущность **Заказ**, списывает товары со склада, вызывает API платежной системы, обновляет статус заказа. |
| **Менеджер каталога** | UC-04 | Форма добавления товара, Сущность **Товар** | Валидирует входящие данные администратора, создает новый экземпляр сущности **Товар** и сохраняет его в базу данных. |
